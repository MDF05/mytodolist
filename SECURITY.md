## Reporting a Vulnerability

We take the security of **MDF Note** seriously. If you believe you have found a security vulnerability in this project, please report it to us as described below.

### 1. Do Not Disclose Publicly
Please **do not** report security vulnerabilities through public GitHub issues, pull requests, or social media. Disclosing vulnerabilities publicly puts the community at risk before a fix can be released.

### 2. How to Report
Please email the maintainers directly at [INSERT EMAIL ADDRESS].
In your email, please include:
-   A description of the vulnerability.
-   Steps to reproduce the issue.
-   Affected versions.
-   Any potential impact.

### 3. Response Timeline
-   **Acknowledgment:** We will acknowledge your report within 48 hours.
-   **Investigation:** We will investigate the issue and communicate our findings within 1 week.
-   **Resolution:** If a vulnerability is confirmed, we will release a patch as soon as possible.

### 4. Credit
We will credit the reporter in the changelog and security advisory (unless you prefer to remain anonymous).

Thank you for helping keep **MDF Note** secure!
